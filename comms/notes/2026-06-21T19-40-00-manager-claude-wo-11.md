## WO-11 — Platform objects + system (after world scaffold)

**Goal.** The 10 authored platforms with reliable landings and **carry behavior** (the player rides
moving platforms without sliding off or sinking). Use the **2d-collision** skill for moving-platform
carry.

**Do (lane: `src/objects/**`, `src/systems/**`).**
- `Platform` object + `PlatformSystem` (or `buildPlatforms(scene, catalog)`). Read every instance from
  `catalog.placements.environmentObstacles` (10 of them) — id, kind, topLeft, brickCount, width,
  height, movement, speed. Compose the visual from the 170×58 brick (`brickCount` repeats); do not
  generate art. Static bodies for `stationary`; kinematic (immovable, body-velocity-driven) for
  `vertical`/`horizontal`.
- Movement: vertical between `movement.min`/`max` on y, horizontal between min/max on x, at `speed`
  px/s, ping-pong. Emit `platform:move-start { platformId, kind, x, y }` when motion begins and
  `platform:turnaround { platformId, kind, x, y, direction }` at each bound.
- **Carry:** when the player stands on a horizontal mover, it must travel with it (no slip); on a
  vertical mover it must rest stably (no sink/jitter). The player controller uses a small
  GROUND_STICK downward velocity and integrates its own gravity (`setAllowGravity(false)`) — account
  for that. Expose the platform bodies so the Level scene wires
  `player.collideWith(platformBodies)`.
- Keep the movement math pure where feasible so a node check asserts ping-pong bounds + turnaround
  direction. Carry/landing feel is browser-verified at WO-16 (`not-run` with reason).

**Acceptance (record in your `result`).**
- `npm run typecheck` + `npm run build` exit 0; a node self-check asserts the 10 instances load from
  the manifest, movers ping-pong within bounds, and turnaround fires with correct direction.
- Exactly the authored bounds/speeds; brick dims from manifest; no hardcoded placements.
- No `src/scenes/**` edits.
