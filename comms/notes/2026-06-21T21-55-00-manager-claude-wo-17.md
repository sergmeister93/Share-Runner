## WO-17 — QA browser smoke test (full continuous playthrough)

**Goal.** Independently verify the first playable end-to-end by **playing it** (not by emitting
events), and record evidence per the `qa-verification` skill (no "looks right" — capture command +
result + screenshots).

**Setup.** `npm install`; `npm run build` (must exit 0). Serve it: `npm run dev` (or `vite preview`
on the built `dist/`). Drive with Playwright/headless Chrome.

> **Important test gotcha (manager-verified):** Phaser **pauses on hidden tab** (default). In headless
> automation the page is often "hidden", which **freezes the game clock** (scene timers/`delayedCall`
> never fire → no scene transitions). Force visibility before driving: override
> `document.visibilityState='visible'` + `hidden=false`, dispatch `visibilitychange` + window `focus`,
> and pump activity. (This is a test-harness concern, not a game bug — a real visible browser runs fine.)

**Verify (record evidence for each):**
- Boot → IntroScene title beat → Menu (Start Game / Quit only).
- Start → Preloader (progress 0→100, assets load, no 404) → Level.
- Player spawns left (~x=220), responds to Arrow/WASD + jump (Up/Space/W); reaches platforms.
- **Collect all 5 shares by traversing/jumping** (incl. the 3 moving-platform coins, E-02). HUD
  counter advances to 5/5, score to 500, counter pulses.
- Touch the flag **before** 5 collected → `flag:locked-feedback`, NO completion (E-03).
- Touch the flag **after** 5 → `level:complete` → score flash → CompletionScene (LEVEL COMPLETE,
  score, restart prompt).
- Restart (Enter/R) → fresh Level at 0/5, score 0, player respawned.
- Music loops after the Start gesture; mute toggle (click + M) silences/restores and **persists**
  across restart (E-05).
- No console errors throughout (the favicon/Phaser-banner lines are fine).

**Acceptance.** A recorded run (screenshots + console) covering the above. Any failure → file a
precise blocker naming the scene/step. Write evidence under `docs/qa/`.
