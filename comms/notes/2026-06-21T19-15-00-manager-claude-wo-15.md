## WO-15 — Music loop + mute toggle (parallel; ux-lane scoped)

**Goal.** Loop the Baltimore rooftop theme during the level and provide a mute toggle that persists.
Build it as self-contained ux-lane units now; I wire them into the real LevelScene at WO-16.

**Manager lane scoping (decision):** for first playable, the audio coordinator lives in the **ux
lane** (`src/ui/**`, e.g. `src/ui/AudioController.ts`) because audio is scene-driven (ux). Do not
create `src/systems/**` files (backend lane) — if you think you need to, raise a `handoff`. GameState
already owns audio truth (`gameState.audio`, frozen); use it, don't duplicate.

**Do (lane: `src/ui/**`, and a scene hook only if needed via handoff).**
- `AudioController` (attachable to a scene's sound manager): on `level:start` (or when attached),
  play the music key `LEVEL_MUSIC_KEY` (`baltimore-rooftop-theme`) looping (manifest
  `loopSuggested:true`), volume `MUSIC_VOLUME`. **Start only after a user gesture** (the Start Game
  click satisfies this) to respect browser autoplay rules. Emit `audio:music-start { musicId, loop,
  volume }`; on stop emit `audio:music-stop { musicId, reason }`.
- Mute toggle UI (small pixel-crisp button + a keyboard shortcut, e.g. `M`). Toggling sets
  `gameState.audio.muted`, applies it to the sound manager, and emits
  `audio:mute-changed { muted, source }`. Mute **persists across reset** — GameState.reset() already
  preserves `audio.muted` (E-05); just read/write that field, don't fight it.
- Tear down listeners/sounds on scene shutdown (restart-safety).

**Acceptance (record in your `result`).**
- `npm run typecheck` + `npm run build` exit 0.
- Browser-verify (attach to a test/stub scene is fine): music loops after a gesture; mute silences and
  unmute restores; the mute state survives a `gameState.reset()` call. Screenshot/console evidence.
- Only frozen audio events; reads music key from Constants/manifest (no hardcoded path); volume from
  Constants.

**Process.** Lane-only (`src/ui/**`); **commit your own** files + ledger in one commit, status
`review`, stop. Do NOT `git add -A` (backend is in `src/objects`/`src/systems`/`src/data`).
